# Bug 批量处理工作流

## 使用方式

### 第 0 步：启动前配置检查（AI 自动做）

**AI 在每次启动流程前，自动检查以下配置：**

| 检查项 | 检查方式 | 是否强制 | 缺失时行为 |
|--------|---------|---------|-----------|
| `TAPD_ACCESS_TOKEN` | 检查环境变量 | ✅ 必须 | 提示配置令牌，流程终止 |
| `workspace_id` | 检查是否为 `YOUR_WORKSPACE_ID` 或空 | ✅ 必须 | 提示填写项目 ID，流程终止 |
| `bot_url` | 检查是否配置 | ❌ 可选 | 仅不发群通知，其他功能正常 |
| `filter_members` | 检查是否为空 | ❌ 可选 | 默认推送所有 Bug |

**示例检查逻辑：**
```
你：收集一下项目里的 Bug
AI：正在检查配置...
    ✅ TAPD_ACCESS_TOKEN: 已配置
    ✅ workspace_id: 39222963
    ⚠️  bot_url: 未配置（跳过群通知）
    → 配置检查通过，开始执行...
```

### 第 1 步：批量收集（AI 自动做）

**你说**：
> "收集一下项目里所有 new 的 Bug，我要批量处理"

**AI 自动做**：
1. 查询 TAPD：`status=new`，获取 Bug 列表
2. **不做自动分类过滤**（由人工判断客户端/服务器）
3. 发群通知：
   ```
   【Bug 批量收集结果】
   共 15 个 new Bug
   
   1. 【任务系统】子任务未置顶
   2. 【背包】出售装备后未获取收益
   3. 【安卓/充值】沙盒充值完成后未到账券
   ...
   
   请回复要处理的 Bug 编号，或说明"跳过第 x 个"
   ```

### 第 2 步：人工确认（你做）

**你说**：
> "修 1、2、5" 或 "修客户端相关的全部" 或 "跳过第 3 个"

### 第 3 步：逐个自动处理（AI 全自动）

**AI 对每个确认的 Bug 自动执行**：

```
【开始处理 Bug 1/5】
标题：【任务系统】子任务未置顶排序
━━━━━━━━━━━━━━━━━━━━

→ 获取详情 + 截图... ✅
→ 解析截图内容... ✅（需求文档：要求可领取任务置顶）
→ 分析代码... ✅（定位到 TaskView.ts 第 163 行）
→ 修改代码... ✅（新增 guestDatas.sort()）
→ 发群通知... ✅
→ 更新 TAPD... ✅（状态：resolved）
━━━━━━━━━━━━━━━━━━━━
【Bug 1/5 完成】

【开始处理 Bug 2/5】...
```

## 人工确认规则

**不做自动分类**，由用户在群里直接指定：

> "修 1、2、5" → 处理指定编号
> "修客户端相关的" → AI 根据标题人工判断
> "跳过第 3 个" → 跳过指定编号
> "全部修" → 处理所有

**注意**：服务器 Bug 也可能使用客户端术语（如"背包无法售卖"），因此不依赖关键词自动过滤，避免误判。

## 每步都发群通知

处理每个 Bug 时，群里会收到：
1. **开始分析** - Bug 标题 + 描述
2. **代码定位** - 问题文件 + 行号 + 原因
3. **修复完成** - 修改内容
4. **TAPD 更新** - 状态变更

不需要人工确认，全自动串行处理。
